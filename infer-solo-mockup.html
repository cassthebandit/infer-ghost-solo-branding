Debugging the Invisible: Nemotron on OpenClaw

Daniel Soteldo 20 Mar 2026 8 min read

The motivation was simple enough: NVIDIA released Nemotron 3 Super, and free inference was available across NVIDIA NIM, OpenRouter, and Kilo. I wanted to wire it into Cass, my Discord AI agent running on OpenClaw, using LiteLLM as a proxy layer.

What followed was a debugging arc that surfaced two silent bugs — the kind that don't throw errors, just quietly produce wrong output. Claude wrote all the code throughout these sessions while I directed architecture decisions, debugging strategy, and verification work.

The model was thinking correctly — then saying nothing, because all the tokens had already been claimed by the reasoning phase.

Bug One: The Silent Role Drop

The first bug: OpenClaw was sending system prompts as role: "developer", which non-OpenAI providers silently drop. No error, no warning — the model just never received its instructions. The fix was two compatibility flags:

# OpenClaw compatibility flags for non-OpenAI providers supportsDeveloperRole: false supportsReasoningEffort: true

Bug Two: The Duplicate Stream

The second bug was subtler. NVIDIA NIM was duplicating reasoning tokens into both reasoning_content and content fields during streaming, causing blank responses in OpenClaw. The model would think through the entire problem correctly, then output nothing — because every token had already been consumed by the reasoning phase.

The fix was a custom ReasoningGuardrail extending LiteLLM's CustomGuardrail, registered as a post_call hook that deduplicates the stream by suppressing content during the thinking phase and suppressing reasoning fields when the answer begins.

# The guardrail class (simplified) class ReasoningGuardrail(CustomGuardrail): async def async_post_call_success_hook(self, data, response): # Detect reasoning phase by checking content fields if response.reasoning_content and response.content: # Suppress duplicate content during thinking response.content = "" return response

The full guardrail code, LiteLLM config, and README are on GitHub at cassthebandit/nemotron-openclaw-fix. The final deployed stack runs four model backends switchable via Discord slash commands, with LiteLLM running as Docker alongside Postgres and Prometheus.
